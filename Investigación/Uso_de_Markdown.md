# Investigación sobre Markdown

## ¿Qué es Markdown y por qué se utiliza en proyectos de software?

Markdown es un lenguaje de marcado ligero que permite dar formato a texto de manera sencilla utilizando caracteres comunes. Es muy utilizado en proyectos de software porque:

* Es fácil de escribir y leer incluso sin ser procesado.
* Permite crear documentación clara y estructurada.
* Se integra perfectamente con plataformas como **GitHub**, **GitLab** y **Bitbucket**.
* Facilita la colaboración entre desarrolladores.

---

## Ejemplo práctico de uso de Markdown

### Encabezados:

# Título 1

## Título 2

### Título 3

### Listas:

**Lista con viñetas:**

* Elemento 1
* Elemento 2
* Elemento 3

**Lista numerada:**

1. Paso 1
2. Paso 2
3. Paso 3

### Tablas

| Luis   | QA  | Chile  |

## Ventajas de utilizar Markdown en combinación con GitHub

* Permite documentar proyectos dentro de los repositorios (**README.md**).
* Se visualiza automáticamente con formato en GitHub.
* Facilita el control de versiones de la documentación con **Git**.
* Compatible con comentarios, wikis y issues de GitHub.
* Ayuda a mantener todo el conocimiento del proyecto en un solo lugar.
